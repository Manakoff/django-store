# 🛒 Store: Полнофункциональная E-commerce платформа на Django

Добро пожаловать в репозиторий проекта **Store**! Это мой второй крупный практический проект, нацеленный на глубокое и всестороннее изучение веб-фреймворка **Django**, построение надежной архитектуры веб-приложений и интеграцию современных backend-технологий. 

Проект разработан с прицелом на коммерческое использование и демонстрирует навыки чистой архитектуры, безопасной авторизации, оптимизации запросов и построения API.

---

## 🛠️ Технический стек проекта
* **Backend:** Python 3.x, Django Web Framework, Django REST Framework (DRF)
* **Базы данных:** PostgreSQL (Production), SQLite (Разработка)
* **Асинхронные задачи & Кэширование:** Celery, Redis
* **Интерфейс & Шаблоны:** Django Templates (HTML/CSS), Работа со статикой и медиафайлами
* **Контроль версий & Workflow:** Git, GitHub, GitFlow (Ветвление)

---

## ⚙️ Методология управления репозиторием (Git)
В проекте строго соблюдается профессиональный подход к ведению контроля версий:
* **`main`** — стабильная ветка. Содержит только проверенный, протестированный и готовый к релизу код.
* **`develop`** — основная ветка разработки. Вся ежедневная работа над задачами, фиксация изменений и промежуточные коммиты происходят именно здесь.
* **Релизный цикл:** Слияние (Merge) ветки `develop` в ветку `main` происходит системно **один раз в неделю** после полного тестирования функционала текущего спринта.

## 📚 План реализации проекта и чек-лист задач

### 🔹 Этап 1: Базовая настройка и архитектура данных
- [+] **1. Начало с проектом:** Инициализация Django-приложения, настройка виртуального окружения, первичная структура директорий, конфигурация базовых настроек.
- [ ] **2. Работа с шаблонами:** Интеграция UI-компонентов, разметка базовых HTML-шаблонов, подключение статических файлов (CSS, JS, изображения).
- [ ] **3. Django ORM, модели и БД:** Проектирование схемы базы данных, создание моделей каталога товаров (категории, продукты), применение миграций.

### 🔹 Этап 2: Бизнес-логика и пользовательский опыт
- [ ] **4. Авторизация и регистрация:** Реализация кастомной системы пользователей, формы входа/регистрации, валидация данных, сессии.
- [ ] **5. Корзина товаров:** Разработка логики добавления, изменения и удаления товаров в корзине с использованием Django Sessions / ORM.
- [ ] **6. Фильтрация, пагинация, Django Admin:** Кастомизация административной панели, добавление фильтров для каталога, разбиение списков товаров на страницы.

### 🔹 Этап 3: Продвинутый Backend и оптимизация
- [ ] **7. CBV, миксины, emails:** Перевод кодовой базы на Class-Based Views (CBV) для переиспользуемости, внедрение миксинов для контроля доступа, отправка email-уведомлений.
- [ ] **8. PostgreSQL, OAuth:** Переход со SQLite на промышленную СУБД PostgreSQL, настройка конфигурации, интеграция авторизации через социальные сети.
- [ ] **9. Тестирование, кеширование, Celery:** Написание Unit и интеграционных тестов, оптимизация запросов через кэширование, вынос тяжелых задач (отправка писем) в асинхронный воркер Celery через Redis.

### 🔹 Этап 4: Финализация и API
- [ ] **10. Заказы и подключение оплаты:** Проектирование моделей заказов, создание страницы оформления, интеграция платежного шлюза.
- [ ] **11. Конфигурирование проекта:** Вынос чувствительных данных в `.env`, разделение настроек на `development` и `production`, финальный рефакторинг кода.
- [ ] **12. Доп. модуль Django REST Framework (DRF):** Построение RESTful API для интеграции со сторонними сервисами или фронтенд-фреймворками, сериализаторы, JWT-авторизация.

## 🚀 Как запустить проект локально

1. Склонируйте репозиторий и перейдите в папку проекта:
   ```bash
   git clone https://github.com
   cd store-server/store
   ```
2. Разверните и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   # Для Windows:
   venv\Scripts\activate
   # Для macOS/Linux:
   source venv/bin/activate
   ```
3. Установите зависимости и примените миграции:
   ```bash
   pip install -r requirements.txt
   python manage.py migrate
   ```
4. Запустите локальный сервер:
   ```bash
   python manage.py runserver
   ```
